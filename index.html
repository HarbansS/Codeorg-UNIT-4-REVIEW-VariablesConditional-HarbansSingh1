Unit 2

In this unit we learned about how the internet works and how we play into it. Computing networks are devices that are interconnected with one another and made so they can receive and send data to one another. Computers can send and receive data by paths which are connections between computers. Paths and computers have bandwidths which is the max amount of data that a computer can send at once. The internet has protocols which are rules set by computers on how they can send data which is from your IP address. Computers have 2 ways of sending data, UDP and TCP. UDP (User Datagram Protocol) is something that will send the data very fast but will not double check to see that all data is sent. TCP (Transmission Control Protocol) is slower than UDP but is a lot more reliable in making sure everything is sent to the correct location. Data is sent in things called packets when they are larger.

Unit 3

Unit 3 is based on the creation of apps. In apps we use everyday there is input and output, input is what you do to the app that gives it information and output is what the computer will show you based on what you put into it. We send information to the computer by a user interface which can let you type, click, shake, and do something to send information. We learn about how specific we need to be for the computer to understand the instructions that we give it. There are 2 ways we can give instructions. Sequential is when the computer reads the code from the beginning to end without any sort of user input. Event driven programming is when the computer gives output based on the user using the app and needing to give some sort of input. Program statements are used to tell the computer what to do. In coding like this bugs are very common so you can do pair programming with another person to check over your work and you can check over theirs.
